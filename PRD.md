# 공학용 전자계산기 웹앱 제품 요구사항 명세서 (PRD)

## 📋 문서 정보
- **프로젝트명**: Scientific Calculator Web App
- **버전**: 1.0
- **작성일**: 2025-12-23
- **최종 수정일**: 2025-12-23

## 1. 제품 개요

### 1.1 제품 비전
현대적이고 직관적인 UI/UX를 갖춘 웹 기반 공학용 계산기를 제공하여, 학생, 엔지니어, 과학자들이 복잡한 수학 계산을 효율적으로 수행할 수 있도록 지원합니다.

### 1.2 제품 목표
- 전문적인 공학용 계산 기능을 웹 브라우저에서 제공
- 모바일부터 데스크톱까지 모든 디바이스에서 일관된 사용자 경험 제공
- 다크 모드 기반의 현대적이고 세련된 디자인
- 빠르고 정확한 계산 결과 제공

### 1.3 타겟 사용자
- **학생**: 공학, 과학, 수학 전공 학생
- **전문가**: 엔지니어, 연구원, 데이터 분석가
- **교육자**: 수학 및 과학 교사
- **일반 사용자**: 복잡한 수학 계산이 필요한 모든 사용자

## 2. 핵심 기능 요구사항

### 2.1 기본 계산 기능

#### 2.1.1 사칙연산
- ➕ 덧셈 (Addition)
- ➖ 뺄셈 (Subtraction)
- ✖️ 곱셈 (Multiplication)
- ➗ 나눗셈 (Division)

#### 2.1.2 숫자 입력
- 0-9 숫자 버튼
- 소수점(.) 입력
- 양수/음수 전환 (+/-)

#### 2.1.3 편집 기능
- **C (Clear)**: 전체 초기화
- **⌫ (Backspace)**: 마지막 입력 삭제
- 실시간 수식 표시

### 2.2 공학 함수

#### 2.2.1 삼각함수 (Trigonometric Functions)
**기본 삼각함수**
- `sin` - 사인 함수
- `cos` - 코사인 함수
- `tan` - 탄젠트 함수

**역삼각함수 (Inverse)**
- `sin⁻¹` (arcsin) - 역사인 함수
- `cos⁻¹` (arccos) - 역코사인 함수
- `tan⁻¹` (arctan) - 역탄젠트 함수

**쌍곡선 함수 (Hyperbolic)**
- `sinh` - 쌍곡선 사인
- `cosh` - 쌍곡선 코사인
- `tanh` - 쌍곡선 탄젠트

**각도 모드**
- **DEG** (Degree): 도 단위
- **RAD** (Radian): 라디안 단위
- 모드 전환 토글 버튼

#### 2.2.2 지수 및 로그 함수
- `x²` - 제곱
- `xʸ` - 거듭제곱
- `√x` - 제곱근
- `exp` - 지수 함수 (e^x)
- `10ˣ` - 10의 거듭제곱
- `ln` - 자연로그 (밑이 e)
- `log` - 상용로그 (밑이 10)

#### 2.2.3 기타 수학 함수
- `|x|` - 절댓값
- `1/x` - 역수
- `n!` - 팩토리얼
- `mod` - 모듈로 연산 (나머지)
- `( )` - 괄호 (연산 우선순위)

#### 2.2.4 수학 상수
- `π` (pi) - 원주율 (3.14159...)
- `e` - 자연상수 (2.71828...)

#### 2.2.5 2차 함수 (2nd Function)
- `2nd` 버튼을 통한 추가 함수 접근
- 버튼의 대체 기능 활성화

### 2.3 사용자 인터페이스 기능

#### 2.3.1 디스플레이 영역
**모드 인디케이터**
- 현재 각도 모드 표시 (DEG/RAD)
- 표시 모드 (FLOAT)
- 상단 좌측에 배치

**수식 표시 영역**
- 입력된 수식을 실시간으로 표시
- 중간 크기 폰트 (lg-xl)
- 회색 계열 색상으로 구분

**결과 표시 영역**
- 계산 결과를 큰 글씨로 표시 (4xl-6xl)
- 애니메이션 커서(|) 표시
- 우측 정렬
- 높은 대비의 텍스트 색상

**복사 기능**
- 결과값 복사 버튼 (📋 아이콘)
- 클릭 시 클립보드에 복사

#### 2.3.2 계산 히스토리
**히스토리 사이드바** (대형 화면 전용)
- 우측 사이드바에 배치
- 이전 계산 기록 표시
- 스크롤 가능한 리스트

**히스토리 항목**
- 입력 수식 표시
- 계산 결과 표시
- 호버 시 하이라이트 효과
- 클릭 시 해당 계산 불러오기

**히스토리 관리**
- "Clear" 버튼으로 전체 삭제
- 자동 스크롤
- 커스텀 스크롤바 디자인

#### 2.3.3 레이아웃 구조
**3-패널 레이아웃**
1. **헤더 (Header)**
   - 앱 로고 및 제목
   - 설정 버튼
   - 히스토리 버튼 (모바일)

2. **메인 영역 (Main)**
   - 디스플레이 화면
   - 버튼 패널 (좌측: 공학 함수, 우측: 삼각함수)

3. **사이드바 (Sidebar)** - 대형 화면만
   - 계산 히스토리

**버튼 패널 구조**
- **좌측 패널**: 5x7 그리드 (공학 함수 + 기본 연산)
- **우측 패널**: 세로 레이아웃 (삼각함수)

#### 2.3.4 반응형 디자인
**브레이크포인트**
- **Mobile**: < 768px
  - 단일 컬럼 레이아웃
  - 히스토리 사이드바 숨김
  
- **Tablet**: 768px - 1024px
  - 2컬럼 레이아웃
  - 히스토리 사이드바 숨김

- **Desktop**: 1024px - 1280px
  - 2컬럼 레이아웃
  - 히스토리 사이드바 숨김

- **Large Desktop**: > 1280px
  - 3컬럼 레이아웃
  - 히스토리 사이드바 표시

#### 2.3.5 테마
**다크 모드** (기본)
- 배경: `#101922`
- 서피스: `#1c2630`
- 서피스 다크: `#283039`
- 텍스트: 흰색

**라이트 모드** (향후 지원)
- 배경: `#f6f7f8`
- 서피스: 흰색
- 텍스트: 어두운 회색

## 3. 기술 요구사항

### 3.1 프론트엔드 기술 스택

#### 3.1.1 핵심 기술
- **HTML5**: 시맨틱 마크업
- **CSS3**: 스타일링 및 애니메이션
- **JavaScript (ES6+)**: 계산 로직 및 인터랙션

#### 3.1.2 프레임워크 및 라이브러리
- **TailwindCSS**: 유틸리티 기반 CSS 프레임워크
  - CDN 방식 사용
  - 다크 모드 설정
  - 커스텀 테마 확장

#### 3.1.3 외부 리소스
- **Google Fonts**: Space Grotesk
- **Material Symbols**: 아이콘 세트

### 3.2 디자인 시스템

#### 3.2.1 색상 팔레트
```css
/* Primary Color */
--primary: #137fec;

/* Background Colors */
--background-light: #f6f7f8;
--background-dark: #101922;

/* Surface Colors */
--surface-dark: #1c2630;
--surface-darker: #283039;

/* Text Colors */
--text-light: #ffffff;
--text-dark: #1e293b;
--text-muted: #64748b;
```

#### 3.2.2 타이포그래피
**폰트 패밀리**
- Primary: Space Grotesk (sans-serif)

**폰트 크기**
- 결과 표시: `text-4xl` ~ `text-6xl` (36px ~ 60px)
- 수식 표시: `text-lg` ~ `text-xl` (18px ~ 20px)
- 버튼 텍스트: `text-sm` ~ `text-base` (14px ~ 16px)
- 모드 인디케이터: `text-[10px]`

**폰트 굵기**
- Light: 300
- Regular: 400
- Medium: 500
- Bold: 700

#### 3.2.3 간격 및 레이아웃
**패딩**
- 컨테이너: `p-4` ~ `p-8`
- 버튼: `p-3`
- 헤더: `px-6 py-4`

**간격**
- 버튼 간격: `gap-3`
- 섹션 간격: `gap-6`

**테두리 반경**
- 기본: `rounded-lg` (0.5rem)
- 카드: `rounded-2xl` (1rem)
- 버튼: `rounded-lg` (0.5rem)

#### 3.2.4 아이콘
- **Material Symbols Outlined**
- 크기: 18px ~ 20px
- 용도: 설정, 히스토리, 백스페이스, 복사

### 3.3 계산 엔진 요구사항

#### 3.3.1 수식 파싱
- 중위 표기법 (Infix Notation) 입력
- 후위 표기법 (Postfix Notation) 변환
- 연산자 우선순위 처리
- 괄호 처리

#### 3.3.2 계산 정확도
- 부동소수점 정밀도: IEEE 754 표준
- 최대 유효 숫자: 15자리
- 오버플로우/언더플로우 처리
- 오류 처리 (0으로 나누기, 정의되지 않은 연산 등)

#### 3.3.3 각도 변환
- DEG ↔ RAD 자동 변환
- 삼각함수 계산 시 각도 모드 적용

### 3.4 성능 요구사항
- **초기 로딩 시간**: < 2초
- **계산 응답 시간**: < 100ms
- **애니메이션 프레임률**: 60fps
- **메모리 사용량**: < 50MB

### 3.5 브라우저 호환성
**지원 브라우저**

- Firefox (최신 2개 버전)

- Edge (최신 2개 버전)

**최소 요구사항**

- Flexbox 지원

## 4. 사용자 경험 (UX) 요구사항

### 4.1 인터랙션 디자인

#### 4.1.1 버튼 피드백
**시각적 피드백**
- **Hover**: 배경색 변경 (`hover:bg-slate-100`)
- **Active**: 스케일 축소 (`active:scale-95`)
- **Transition**: 부드러운 전환 효과 (200ms)

**특수 버튼**
- **등호(=) 버튼**: 
  - Primary 색상 배경
  - 그림자 효과 (`shadow-lg`)
  - 더 큰 크기 강조

#### 4.1.2 애니메이션
- **커서 애니메이션**: 깜박이는 커서 (`animate-pulse`)
- **버튼 클릭**: 스케일 애니메이션
- **히스토리 항목**: 호버 시 배경색 전환
- **모드 전환**: 부드러운 색상 전환

#### 4.1.3 키보드 지원
- 숫자 키 (0-9)
- 연산자 키 (+, -, *, /)
- Enter: 계산 실행
- Backspace: 삭제
- Escape: 초기화

### 4.2 접근성 (Accessibility)

#### 4.2.1 WCAG 2.1 준수
- **AA 등급** 색상 대비
- 키보드 네비게이션 지원

#### 4.2.2 시각적 접근성
- 고대비 색상 조합

### 4.3 사용성

#### 4.3.1 직관적 레이아웃
- 일반 계산기와 유사한 버튼 배치

- **클라이언트 사이드**: 서버 없이 브라우저에서만 동작
- **브라우저 의존성**: 최신 브라우저 기능 필요

### 8.2 가정
- 사용자는 기본적인 수학 지식 보유
- 사용자는 현대적인 웹 브라우저 사용
- 인터넷 연결 필요 (CDN 리소스 로드)
